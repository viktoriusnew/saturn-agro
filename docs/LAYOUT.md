# Рекомендации по вёрстке и центрированию контента

> **Назначение**: Использовать при изменениях блоков или редактировании контента, чтобы сохранить единообразие и корректное центрирование.

---

## Проблема с `section-padding` и `100vw`

Класс `section-padding` в `globals.css` использует формулу `calc((100% - 1200px) / 2)` для горизонтальных отступов. **Важно:** использовать `100%`, а не `100vw`, т.к. `100vw` включает ширину скроллбара и вызывает визуальное смещение контента влево.

В `html` добавлен `scrollbar-gutter: stable` — резервирует место под скроллбар, чтобы центрирование было симметричным.

---

## Рабочий подход для центрированного блока (раздел 1)

Для секций, где весь контент должен быть строго по центру (заголовок, таймлайн, текст):

1. **Не использовать** `section-padding` на секции, если он даёт конфликты.
2. **Секция:** `paddingLeft: "5%", paddingRight: "5%"` — симметричные отступы от краёв.
3. **Единый контейнер:** `maxWidth: "900px", marginLeft: "auto", marginRight: "auto"` — весь контент внутри одного блока, центрированного по горизонтали.
4. **Inline-стили** для критичных свойств — избегают конфликтов с Tailwind и гарантируют предсказуемое поведение.

### Пример структуры (Section01General)

```tsx
<section style={{ paddingLeft: "5%", paddingRight: "5%" }}>
  <div style={{ maxWidth: "900px", marginLeft: "auto", marginRight: "auto" }}>
    <SectionHeading center />
    {/* таймлайн */}
    {/* текст */}
  </div>
</section>
```

## Остальные секции

Секции 2–10 и контакты используют `section-padding` из `globals.css`. При добавлении новых блоков или изменении существующих:

- Если контент смещается — проверить, не переопределяет ли Tailwind или другой CSS `margin`/`padding`.
- При необходимости применить тот же подход, что в разделе 1: `padding: 0 5%` + `max-width + margin: auto` на внутреннем контейнере.

## Деплой

```bash
cd /root/saturn-agro && npm run build && systemctl restart saturn-agro
```

При изменениях CSS — полная пересборка с очисткой кэша:
```bash
rm -rf .next && npm run build && systemctl restart saturn-agro
```

После деплоя пользователю — обновить страницу с **Ctrl+Shift+R** или открыть в инкогнито, чтобы избежать кэша браузера.
